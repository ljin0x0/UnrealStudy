오늘은 C 언어 프로그램이 실행되는 전체 흐름과 개발 환경 사용법, 그리고 기본적인 프로그램 구조와 데이터 표현 개념을 학습했다.

---

## 핵심 개념 정리

### 코딩과 언어 구조

**코딩**

* 컴퓨터가 이해 가능한 명령서를 작성하는 과정

### 언어의 종류

* 기계어 → 컴퓨터가 바로 이해하는 2진법 언어
* 어셈블리어 → 기계어를 사람이 읽기 쉽게 문자로 표현한 저급 언어
* 고급언어 → C, C++, Java, Phyton 등 사람이 이해하기 쉽게 만든 언어

**소스코드**

* 프로그래머가 고급 언어로 작성한 코드

---

### 컴파일과 빌드

**컴파일**

* 소스코드를 컴퓨터가 이해 가능한 코드로 변환하는 과정
* 소스코드부터 오브젝트 코드까지의 과정

**컴파일러**

* 소스코드로 실행파일까지 만들어 주는 프로그램
* 고급 언어로 작성한 명령서를 기계어로 변환해주는 프로그램

**빌드 프로세스**

* 컴파일 + 링킹

---

### Visual Studio 구조

* 솔루션 → 프로젝트 → 소스코드 구조로 관리됨 

폴더 구조와 유사한 계층 형태로 구성되어 있음

```
Solution
 └ Project
     └ Source Files
```

### 코드

**main() 함수**

* 프로그램의 시작점(Entry Point)

**printf() 함수(print formatted)**

* 양식에 맞게 출력하다라는 뜻

**탈출 문자열(Escape Sequence)**

* 탈출 문자 ’\’와 함께 작성된 문자열
* 문자열내부에서 특수 기능 수행
* ex) \n → 줄바꿈(New line) 

**서식 지정자(Format Specifier)**

* printf() 혹은 scanf() 등 입출력 함수들과 함께 쓰임
* 문자열의 원하는 위치에 넣어 사용

**리터럴(Literal)**

* 소스코드에 적힌 값 그 자체

**자료형(Type)**

* 데이터를 저장하고 처리하기 위해 값의 종류와 크기를 정해 놓은 타입

## 새로 알게 된 점

* Visual Studio가 솔루션 → 프로젝트 → 소스코드 구조로 관리된다는 점을 이해했다.
* 소스코드가 바로 실행되는 게 아닌 컴파일 과정을 거쳐 실행된다는 것을 알게 됐다.
* main() 함수가 프로그램 실행의 시작점이라는 것을 알게 됐다.
* 자료형이 데이터를 저장할 때 값의 종류와 크기를 결정한다는 점을 이해했다.
