#Unreal 게임개발종합반 1-1 - 1-3

###Unity 와 Unreal 차이점
    -Unity는 C#, Unreal은 C++을 사용
    -Unity는 모바일 멀티 플랫폼 지원이 강력해 인디게임이나 모바일 게임에 사용됨
    -Unreal은 AAA급 게임, 고사양 게임에 많이 사용
    포트나이트, 기어즈 오브 워 등 하이엔드 게임에 사용됨
###서버란?
    -게임의 핵심 로직을 담당하며 여러 명의 플레이어가 동시에 접속해도 공정하고 일관성 있게 게임 진행시키기 위해 필요
    웹서버
    -정적인 콘텐츠를 전송하는데 사용되지만 동적인 콘텐츠도 처리가능
    -사용자의 브라우저 요청 http, https 에 맞춰 웹 페이지나 이미지, 데이터를 제공
    게임서버
    -웹 서버와는 다르게 실시간 상호작용과 복잡한 데이터를 다룸
    -많은 사용자의 상태와 게임 데이터를 동시에 처리해야해 서버 성능이 중요하며, 일반적으로 웹 서버보다 복잡한 시스템을 갖춤
###프로토콜이란?
    -컴퓨터나 전자기기 간에 정보를 교환할 때 따르는 규칙의 집합을 말함
    -통신규약이라고도 불리며 컴퓨터 간의 원활한 통신을 위해 지켜지는 약속
    -클라이언트와 서버가 통신하려면 반드시 프로토콜이라는 약속이 필요
    HTTP/HTTPS - 웹에서 쓰이며 요청/응답 구조가 명확해, REST API등 통해 데이터 전송
    TCP/UDP - 게임에서는 실시간성이 중요한 경우 UDP 사용하기도 하며, 안정성이 중요한 경우 TCP 사용
    기타 - WebSocket (브라우저 실시간 통신)
        - RUDP (신뢰성을 높인 UDP)
###클라이언트
    -실제로 화면에 게임을 보여주고, 사용자의 키보드/마우스/터치 입력을 받아 서버에 전달하며 서버에서 받은 결과를 화면에 그래픽이나 사운드로 표시
###P2P(Peer to Peer)
    -중앙 서버 없이, 플레이어끼리 직접 연결해 게임 데이터를 주고받는 방식이며, 서버 비용은 적지만 해킹 방어나 속도 문제가 발생하기 쉬움
###하이브리드
    -부분적으로는 서버가, 부분적으로는 플레이어끼리 통신도 하는 형태이며 다양한 온라인 게임이 상황에 따라 혼합해 사용하기도 함
###단일서버
    -한 대의 서버가 게임의 모든 기능을 책임지는 구조이며, 모든 데이터를 한 곳에서 처리하고 모든 사용자가 이 한 서버에 접속하는 방식
    -인원이 적을 땐 효율적이지만, 사용자 수가 많아지면 느려지고 서버에 과부하가 걸릴 수 있음
###분산서버
    -여러 서버가 각각 역할을 나눠 처리하는 방식이며 로그인, 채팅, 전투 기능 등 서버가 맡은 역할이 나눠져 있어 분산된 구조로 운영됨
    -사람들이 몰려도 부하가 덜 걸리며 역할을 나눠 효율적으로 운영할 수 있어 대규모 온라인 게임에 적합
###의사결정권자
    PD(Project Director) - 한 팀의 헤드 역할이며, 게임 전반의 방향성을 결정
    TD(Technical Director) - 서버팀, 클라이언트팀 등 모든 기술팀의 헤드 역할이며, 전체적인 기술의 흐름 및 방향성을 결정
    AD(Art Director) - 아트팀의 모든 방향성을 결정하고 3D 그래픽, 2D 컨셉 등 가능성 여부 결정



##퀘스트
###1. Unreal Engine을 조사해요!
게임엔진이란?
    -비디오 게임이나 실시간 인터페이스를 쉽고 빠르게 개발할 수 있도록 그래픽, 물리, 사운드, 입력 등 핵심 기술을 통합해 제공하는 저작 도구 소프트웨어
###Unreal Engine이란?
    -에픽게임즈가 개발해 실시간 3D제작 도구 및 게임 엔진으로, 고품질 그래픽과 다양한 개발 기능을 제공해 게임뿐만 아니라 영화, 건축, 시뮬레이션 등 폭넓은 분야에서 활용
###Unreal Engine의 주요 기능(그래픽 렌더링, 물리 엔진, 비주얼 스크립팅 등)을 간략하게 설명해주세요. 내가 하는 게임으로 예시를 들어주신다면 베스트 입니다!
    예시 게임 : 발로란트
    그래픽 렌더링 - 스킬 이펙트가 명확히 보이도록 색 대비와 윤곽을 강조
    물리 엔진 - 벽에 스킬이 튕기거나 퍼지는 효과나 총알이 맞는 판정(히트박스)
    비주얼 스크립팅 - 맵의 문 열림이나 캐릭터의 스킬 로직 설계
###Unreal Engine으로 제작된 대표적인 게임 2~3개를 조사하고, 해당 게임에서 Unreal이 어떤 역할을 했는지 분석하세요.
    대표적 게임 : 포트나이트, 배틀그라운드, 파이널 판타지 7 리메이크
    포트나이트
        -Unreal Engine의 차세대 그래픽 기술을 적용해 높은 그래픽 품질과 안정적인 프레임을 동시에 확보
        -100명이 동시에 접속하는 배틀로얄 구조 속에서도 건축 시스템과 전투 상황이 실시간으로 동기화되도록 구현 이는 Unreal Engine의 멀티플레이 네트워크 프레임워크와 서버-클라이언트 구조 덕분
    배틀그라운드
        -Unreal Engine의 오픈월드 관리 능력과 물리 시스템을 잘 보여주는 게임
        -광활한 전장을 실시간으로 로딩하고 처리해야 하므로, 월드 스트리밍과 LOD 시스템이 중요하게 작용
        -총기, 차량 이동, 충돌 판정과 같은 요소들은 정교한 물리 연산과 충돌 처리 기능에 기반
        -100명의 플레이어가 동시에 전투를 벌이는 환경에서 이러한 데이터가 정확히 동기화 되도록 하는 것 역시 Unreal Engine의 네트워크 기능이 중요한 역할을 함
    파이널판타지 7 리메이크
        -시네마틱 연출과 캐릭터 표현에서도 강력한 도구임을 보여주는 사례
        -실시간 렌더링 기반의 컷신과 게임 플레이를 자연스럽게 연결하고, 고품질 캐릭터 모델, 피부 표현, 의상과 머리카락의 디테일은 Unreal Engine의 렌더링 기술을 기반으로 함
        -전투 장면에서도 다양한 이펙트와 애니메이션 상태 전환이 자연스럽게 이루어지며, 이는 엔진의 애니메이션 시스템과 그래픽 파이프라인 덕분
##2. 게임 레벨(Level)의 개념 탐구
###레벨은 게임의 구성 요소에서 어떤 역할을 하는지 설명해주세요.
    -게임 진행 구조를 형성하는 단위
    -플레이어는 각 레벨을 순차적으로 클리어하며 게임의 목표에 접근하기에 레벨은 게임의 시간적/공간적 구분을 담당하며 전체 스토리와 진행 흐름을 조직하는 역할
    -난이도 조절의 핵심 수단으로 적의 배치, 장애물 구조, 아이템 위치 등 모두 레벨 설계를 통해 결정
    -스토리 전달의 매개체 역할을 하는데 RPG나 어드벤처 게임에서는 환경 오브젝트, 배경 구조, 지역의 변화 등을 통해 세계관과 서사를 전달
###Unreal Engine에서 사용하는 게임 레벨(Level)의 주요 구성 요소(World, Actor, Light, Camera 등)를 조사하고, 각 요소의 역할을 간략하게 설명해주세요.
    World - 레벨 전체를 관리하는 최상위 개념
          - 모든 Actor가 존재하는 공간이며, 물리 연산, 시간 흐름, 게임 상태 등의 전반적인 환경을 통제
          - 레벨을 담고 있는 기본 틀이라고 볼 수 있음
    Actor - 레벨 안에 배치되는 모든 객체의 기본 단위
          - 캐릭터, 아이템, 건물, 카메라, 조명 등 대부분의 요소가 Actor로 구성
          - 위치, 회전, 크기 등의 정보를 가지며 게임 내 상호작용의 중심이 됨
    Light - 장면의 밝기와 분위기를 결정하는 요소
          - Directionjal Light, Point Light, Spot Light 등 다양한 종류가 있고 그림자 생성과 반사 효과를 통해 공간의 사실성과 몰입도를 높임
    Camera - 플레이어가 화면을 통해 바라보는 시점을 담당
           - 1인칭, 3인칭, 고정 카메라 드 다양한 형태로 구현되며, 플레이 경험과 연출 방식에 직접적인 영향을 줌
